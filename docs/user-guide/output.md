# Understanding the Output

## Analysis panel

The analysis is displayed in a cyan-bordered panel titled
**Job Analysis — \<filename\>**.  It is structured Markdown with the
following sections:

- **Root Cause** — a concise statement of why the job failed, or
  confirmation that it succeeded.
- **Failure Type** — infrastructure (hardware, network, filesystem) or
  software/model (bad configuration, missing input, model crash).
- **Error Messages** — the exact error strings from the log that indicate
  the failure.
- **Segment / Simulation Date** — the segment number and model date at
  which the failure occurred, where applicable.
- **Recommended Next Steps** — concrete, user-actionable steps.  These
  are tailored to what the user can actually do (no admin-only operations).

## Help desk ticket panel

Shown only when `--ticket` is passed.  Displayed in a yellow-bordered panel
titled **Suggested Help Desk Ticket**.  Contains:

- **Subject** — a single line with the cluster name and a brief description.
- **Body** — four paragraphs:
    1. Job identification (name, ID, cluster, partition, workflow, stdout path,
       start/end times)
    2. Error summary (exact error strings, segment number, implicated nodes)
    3. Node/infrastructure details (full allocated node list)
    4. Request to administrators

A reminder is printed below the ticket panel to review and add any additional
context before submitting — for example, whether the failure has occurred
before, or steps already taken.

## AI disclaimer

Every run ends with:

> *This report was generated by an AI assistant.  The information provided
> may be incomplete or incorrect.  Please verify all findings and suggestions
> before taking action.*

## Using `--context`

The `--context` / `-c` flag lets you pass information the AI cannot infer
from the log alone.  This affects both the analysis and the ticket.  Examples:

```bash
# Report a known-bad node
analyze-job myjob.o135797758 -c "node n0447 has failed twice this week"

# Note that this is a repeated failure
analyze-job myjob.o135797758 -c "this is the third time this job has failed with the same error"

# Provide experiment context
analyze-job myjob.o135797758 -c "this is a 50-year production run, segment 93 of 600"
```

Information passed via `--context` is included verbatim in the help desk
ticket when relevant — this is the correct way to convey whether a failure
is new or repeated.

## Raw output

`--raw` prints plain Markdown to stdout instead of rendered panels.
Useful for saving to a file, piping to another tool, or including in a
report:

```bash
analyze-job myjob.o135797758 --raw > analysis.md
analyze-job myjob.o135797758 --ticket --raw > full-report.md
```
