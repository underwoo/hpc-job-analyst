# Quick Start

## Prerequisites

- Python 3.11 or newer (via conda or system Python)
- Access to an HPC system running Slurm
- A USAi API key (contact your HPC center)

## Install

```bash
# Clone the repository
git clone https://github.com/SethUnderwood/hpc-job-analyst.git
cd hpc-job-analyst

# Create the conda environment and install in editable mode
conda env create -f environment.yml
conda activate hpc-job-analyst
```

## Start the proxy

The proxy holds your API key and forwards requests to the AI endpoint.
It runs as a background process on your login node.

```bash
analyze-job proxy install \
  --proxy-script /path/to/hpc-job-analyst/server/proxy.py \
  --env-file ~/.config/hpc-job-analyst/env

# Follow the printed instructions, then verify:
analyze-job proxy status
```

See the [Installation guide](../admin-guide/installation.md) for full details.

## Analyze a job

```bash
# Basic analysis
analyze-job /path/to/myjob.o135797758

# With extra context (e.g. a known-bad node)
analyze-job /path/to/myjob.o135797758 --context "node n0447 has been intermittent"

# Include a draft help desk ticket
analyze-job /path/to/myjob.o135797758 --ticket

# Raw Markdown output (for piping or saving)
analyze-job /path/to/myjob.o135797758 --raw > analysis.md
```

## Output

The tool prints a structured analysis panel followed by an AI disclaimer.
If `--ticket` is used, a draft help desk ticket is printed below the analysis
along with a reminder to review and supplement it before submitting.

!!! note "AI-generated output"
    All output is generated by an AI assistant and may be incomplete or
    incorrect.  Verify all findings before taking action.
